Maven Scope 详解

2020.04.08

Maven Scope 包含 compile、test、runtime、provided、system、import。其中默认依赖配置项是 compile。那么这几个配置有什么区别，我们来详解一下。

compile

不显示指定 Scope 时，作为默认选项。compile 表示被依赖的项目需要参与当前新项目的编译，测试，运行及打包。是较强的 scope 选项。

test

需要显示指定，test 表示被依赖的项目仅仅参与测试工作，包括测试代码的编译和执行。

runtime

需要显示指定，runtime 表示被依赖的项目不需要参与项目的编译，但需要参与测试和运行。相比较于 compile，跳过了编译阶段。通常需要和 optional 搭配使用，optional 为布尔值，例如 API 配置为 compile， impl 配置为 runtime，可以选用不同的具体实现。

provided

需要显示指定，provided 表示被依赖的项目不需要被打包进去，参与编译、测试、运行等阶段。

system

需要显示指定，system 基本和 provided 一致，区别在于被依赖项不会从 maven 仓库拉取，而是从本地文件系统获取，需要配合 systemPath 属性使用。

import

一般只在 dependencyManagement 中使用，表示从其它的 pom 中导入 dependency 的配置。
